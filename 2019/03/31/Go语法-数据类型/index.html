基本类型

UTF-8

早期 美国用ASCll字符集，但是其他国家无法使用自己的符号系统，混合多种语言的数据Unicode产生，每个符号为int32,但是浪费空间，utf-8产生。

UTF8是一个将Unicode码点编码为字节序列的变长编码，UTF8编码使用1到4个字节来表示每个Unicode码点，

ASCII部分字符只使用1个字节
常用字符部分使用2或3个字节表示

每个符号编码后，第一个字节的高端bit为：
0 表示对应7bit的ASCII字符，ASCII字符每个字符依然是一个字节，和传统的ASCII编码兼容。
110 说明需要2个字节；后续的每个高端bit都以10开头

变长的编码无法直接通过索引来访问第n个字符
UTF8编码比较紧凑，完全兼容ASCII码，并且可以自动同步：它可以通过向前回朔最多2个字节就能确定当前字符编码的开始字节的位置
它也是一个前缀编码，所以当从左向右解码时不会有任何歧义也并不需要向前查看
没有任何字符的编码是其它字符编码的子串，或是其它编码序列的字串，因此搜索一个字符时只要搜索它的字节编码序列即可，不用担心前后的上下文会对搜索结果产生干扰

字符串和Byte切片

标准库中有四个包对字符串处理尤为重要：bytes、strings、strconv和unicode包
strings包提供了许多如字符串的查询、替换、比较、截断、拆分和合并等功能。
strconv包提供了布尔型、整型数、浮点数和对应字符串的相互转换，还提供了双引号转义相关的转换。
unicode包提供了IsDigit、IsLetter、IsUpper和IsLower等类似功能

数组

  • 数组的长度是根据初始化值的个数来计算

-打印出数组的值

  • 指针数组

指针数组和数组本身差不多，只不过元素类型是指针。

只可以给索引1和3赋值,其他赋值需要先分配内存。

  • 函数间传递数组

以值的方式，如果变量是个数组，那么就会整个复制，并传递给函数。当数组很大时，传递数组的指针。

长度和类型一样是可以赋值的

Slicce(切片)

变长的序列，序列中每个元素都有相同的类型,

slice由三个部分构成：指针、长度和容量

len和cap函数分别返回slice的长度和容量。

切片并不存储任何数据，它只是描述了底层数组中的一段

  • make创建切片

和数组的区别是 数组[]内要有值，而切片[]不需要。

  • nil切片表示不存在的切片，而空切片表示一个空集合
  • 切片多用于基于现有的数组或者切片创建
  • 内置append函数可以为一个切片追加一个元素
  • 使用for range遍历

忽略索引或值，使用_

切片和原切片共用的是一个底层数组，值修改，底层数组的值就会被改变

函数间传递切片 原切片和新切片地址不一样，切片在函数间传递是复制的。修改一个索引的值后，发现原切片的值也被修改了，说明它们共用一个底层数组。

Map

是一个集合，用于存储一系列无序的键值对(key/value)。

  • 但是map中的元素并不是一个变量，因此我们不能对map的元素进行取址操作

禁止对map元素取址的原因是map可能随着元素数量的增长而重新分配更大的内存空间，从而可能导致之前的地址无效

  • 遍历map

遍历的顺序是随机的，每一次遍历的顺序都不相同.

map的key中不存在value 将返回零值。当类型是数字时，就要使用map的下标语法产生的第二个值，判断是否存在了。

  • 判断相等

需要比较长度，和值

  • 在函数间传递map

传递的是本身，函数内修改了，会影响到原map
