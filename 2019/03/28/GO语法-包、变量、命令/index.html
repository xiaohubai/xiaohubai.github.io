包

  • 每个 Go 程序都是由包组成的。程序运行的入口是包 main。

    导入

导入的包名称重复，对导入的包重新命名（我们导入的时候，在包名的左侧，起一个新的包名就可以了。）

导入的包原则上必须使用，如果导入后暂时不用 前面加空白标志符 “_”（使用_重命名我们导入的包）

下划线 “_”是空标识符，可以当作临时变量，因为go 不允许存在无用的临时变量，不然编译错误。

导出

  • 在 Go 中，首字母大写的名称是可以被导出的，也就是可以被别的包调用，小写的不行。

包的init函数

  • 每个包都可以有任意多个init函数，这些init函数都会在main函数之前执行，init函数通常用来做初始化变量、设置包或者其他需要在程序执行前的引导工作。
  • 比如上面我们讲的需要使用_空标志符来导入一个包的目的，就是想执行这个包里的init函数（比如数据库驱动加载）

go 终端命令

  • test 单元测试

写有单元测试的文件名，必须以_test.go结尾

测试文件要包含若干个测试函数。

这些测试函数要以Test为前缀，还要接收一个*testing.T类型的参数。

常量

  • iota 常量生成器

常量可以是字符、字符串、布尔值或数值。

常量不能用 := 语法声明。

常量的运算都可以在编译期完成，这样可以减少运行时的工作，也方便其他编译优化

类型声明

Celsius和Fahrenheit分别对应不同的温度单位,相同的底层类型float64，但是它们是不同的数据类型，因此它们不可以被相互比较或混在一个表达式运算。刻意区分类型，可以避免一些像无意中使用。

作用域

当编译器遇到一个名字引用时，如果它看起来像一个声明，它首先从最内层的词法域向全局的作用域查找。内部声明屏蔽了外部同名的声明，让外部的声明的名字无法被访问。
