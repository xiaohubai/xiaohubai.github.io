Go语法-函数、方法

2019-03-28

函数

  • 函数声明包括函数名、形式参数列表、返回值列表（可省略）以及函数体

当两个或多个连续的函数命名参数是同一类型，则除了最后一个类型之外，其他都可以省略。

在Go语言中，函数是指不属于任何结构体、类型的方法，也就是说，函数是没有接收者的；而方法是有接收者的，我们说的方法要么是属于一个结构体的，要么属于一个新定义的类型的。（无所属）

实参通过值的方式传递，因此函数的形参是实参的拷贝。对形参进行修改不会影响实参。但是，如果实参包括引用类型，如指针，slice(切片)、map、function、channel等类型，实参可能会由于函数的简介引用被修改。

多返回值

Go 的返回值可以被命名，并且像变量那样使用。没有参数的 return 语句返回结果的当前值。

返回值的多参是有意义的，所有的返回值都应该包含错误信息。

错误处理

在Go中有一部分函数总是能成功的运行，但是大部分不能保证成功运行。对于那些将运行失败看作是预期结果的函数，它们会返回一个额外的返回值，通常是最后一个，来传递错误信息。

可以是一个布尔值 ok, 也可能是复杂的错误信息，error类型：error类型可能是nil或者non-nil。nil意味着函数运行成功，non-nil表示失败。

错误处理策略五种

  • 当错误产生时，函数已经做了一些事情，而这些有助于我们错误的处理，函数会构造一个错误信息返回给调用者。

  • 如果错误的发生是偶然性的，或由不可预知的问题导致的。一个明智的选择是重新尝试失败的操作。在重试时，我们需要限制重试的时间间隔或重试的次数，防止无限制的重试。

  • 输出错误信息并结束程序，但只应在main中执行。对库函数而言，应仅向上传播错误。

  • 可以直接忽略掉错误

  • 只需要输出错误信息就足够了，不需要中断程序的运行。可以通过log包提供函数。

  • 函数像其他值一样，拥有类型，可以被赋值给其他变量
  • 函数类型的零值是nil。调用值为nil的函数值会引起panic错误：

匿名函数

函数值字面量是一种表达式，它的值被称为匿名函数（anonymous function）：func关键字后没有函数名。

函数字面量允许我们在使用函数时，再定义它。

匿名函数可以访问和更新有函数名的局部变量（变量的生命周期不由它的作用域决定），这意味者匿名函数中存在变量引用，这就是函数值属于引用类型和函数值不可比较的原因。

可变参数

在最后一个参数类型之前加上省略符号“…”

定义可变参数(类型不确定，会导致接收函数的处理复杂)

接受可变参数的函数（range..）range 表达式（数组、指向数组的指针、切片、字符串、map、可以接收传输的 channel）

如果原始参数已经是切片类型，只需在最后一个参数后加上省略符。

Deferred函数

在调用普通函数或方法前加上关键字defer，defer语句被执行时，跟在defer后面的函数会被延迟执行。直到包含该defer语句的函数执行完毕时，defer后的函数才会被执行，不论包含defer语句的函数是通过return正常结束，还是由于panic导致的异常结束。

defer语句经常被用于处理成对的操作，如打开、关闭、连接、断开连接、加锁、释放锁。通过defer机制，不论函数逻辑多复杂，都能保证在任何执行路径下，资源被释放。

Panic异常

Go的类型系统会在编译时捕获很多错误，但有些错误只能在运行时检查,这些运行时错误会引起painc异常.发生panic时的函数调用堆栈跟踪信息.

接调用内置的panic函数也会引发panic异常,当某些不应该发生的场景发生时，我们就应该调用panic.

Recover捕获异常

通常来说，不应该对panic异常做任何处理，但有时，也许我们可以从异常中恢复.使用 defer

方法

在定义的时候，会在func和方法名之间增加一个参数，这个参数就是接收者。（有所属）

Go 没有类。不过你可以为结构体类型定义方法。

方法就是一类带特殊的接收者 参数的函数。

接收者的类型定义和方法声明必须在同一包内；不能为内建类型声明方法。

也可以为非结构体类型声明方法。

基于指针对象的方法

当参数实在太大，避免默认的这种拷贝，我们这时候就需要使用指针

值接收者使用的是值的副本来调用方法，而指针接收者使实参的值来调用方法。

由于方法经常需要修改它的接收者，指针接收者比值接收者更常用。但方法的值接收者，可以起到封装作用。

  • 去除方法的函数形式
  • 带指针参数的函数必须接受一个指针。
  • 以指针为接收者的方法被调用时，接收者既能为值又能为指针。
  • 通过嵌入结构体来扩展类型

我们可以直接认为通过嵌入的字段就是ColoredPoint自身的字段，而完全不需要在调用时指出Point

封装

一个对象的变量或者方法如果对调用方是不可见的话，一般就被定义为“封装”

大写首字母的标识符会从定义它们的包中被导出，小写字母的则不会。这种限制包内成员的方式同样适用于struct或者一个类型的方法

隐藏实现的细节，可以防止调用方依赖那些可能变化的具体实现，这样使设计包的程序员在不破坏对外的api情况下能得到更大的自由。

阻止了外部调用方对对象内部的值任意地进行修改
