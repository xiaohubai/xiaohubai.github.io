Go语法-接口

2019-04-13

接口

接口是一种抽象的类型，只有一组接口方法，不知道内部实现，可以利用提供的方法做什么。

接口内很多方法，既然是接口就应该是共有服务，那么每一个对象类型，都可以调用它，但是对象的类型不一样，就需要对象的方法参数为 接口的接口名。方法内 才能调用接口方法。

类型通过实现一个接口的所有方法来实现该接口。

接口值

接口也是值，也可以向其它值一样传递。可用于函数的参数或返回值。

  • 接口值 为 野指针。

即具体类型被传递为野指针时，不会触发野指针异常，而会写一些方法处理。

  • nil接口值

即调用接口后，不给接口具体的值。

nil接口值不保存值也不保存具体类型。

会产生运行时错误。因为接口的元组内并未包含能够指明该调用哪个 具体 方法的类型。

空接口

零个方法的接口值。

可以保存任何类型的值，用来处理未知类型的值。

类型断言

根据 接口内的类型，提供了直接访问接口值底层具体值的方式。

类型选择

按顺序从几个类型断言中选择类型，case 为类型，case内为各个类型的独特操作。

错误

error 类型是一个内建接口
error 为 nil 时表示成功；非 nil 的 error 表示失败。
