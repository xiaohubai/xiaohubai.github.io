参数传递与返回值：

函数后 加const:不改变数据内容。

c1.imag() 错误！编译器认为imag()没加const,会改变数据内容，而我们实例化的c1只是读取数据。

尽量所有的参数传递都传引用。

引用传递后，别名修改内容后会影响引用传递前的实体(引用的特性)，让，不能修改传递的内容。

友元：

当class类将数据private封装后，类外不能直接动class内的数据，有一种例外：类的友元。

相同class的各个object互为friends(友元)

代码质量：

数据一定放在private内

参数尽量以引用方式传递，加const看情况。

返回值尽量用引用方式传递。

class body内，应该加const就加，不然有时会报错

构造函数尽量函数体外初始化数据。

return by reference

函数内有两个参数a,b。函数作用是计算a+b,但是a+b的值放在那里？所以函数内创建变量c存入a+b的结果，但是这时候函数返回不能是c的引用，因为c随着函数销毁而销毁。

传递者无需知道接收者是以引用形式接收。

操作符重载：

  • 语法形式上：只是r指向了c1.

  • 理解实际上：有一个隐含的this指针，指向的是c2。

类中 只要有指针，就尽量操作符重载

所有的指针操作之前，必须指针检查

  • 拷贝赋值函数：

s2=s1;先清除s2的空间，再创建和s1一样的空间结构，再执行赋值。但是如果是浅拷贝,s1和s2指针指向同一个空间，不能先清除s2，所以先判断:

清除s2;创建和s1一样的空间；执行赋值；
